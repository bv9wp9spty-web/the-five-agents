# yuval (תקיית עבודה)

**נתיב:** `yuval/`
**בעלים:** [[ראובן]] (או המשתמש) מוסיף לתקיית `reference/` תמונות השראה; [[יובל]] (הסוכן) קורא אותן ויוצר outputs.
**סטטוס:** **פעילה — אך ריקה.** התקייה הוקמה עם `.gitkeep` בכל תת-תקייה. תמונות reference ו-outputs יצטברו עם הזמן.

> ⚠️ הקובץ הזה מתעד את **התקייה** `yuval/`. אל תבלבלו בינו לבין [[יובל]] (דף הסוכן עצמו תחת `40 - סוכנים/`).

## מה התקייה מכילה

שתי תת-תקיות:

- **`yuval/reference/`** — תמונות השראה שמגדירות את הסגנון הוויזואלי של הצוות. [[ראובן]] או המשתמש מוסיף ידנית. יכול לכלול גם sibling `.txt` לכל תמונה עם תיאור/prompt — כדי שיובל יוכל לקרוא טקסטואלית את הסגנון. **טרם מולאה.**
- **`yuval/outputs/`** — תמונות מוגמרות שיובל יוצר. כל תמונה נשמרת בפורמט `<YYYY-MM-DD>-<slug>.png`, וצמוד לה sibling `<YYYY-MM-DD>-<slug>.txt` שמכיל את ה-prompt ששימש (לאיטרציה).

## Flow השימוש

בכל בקשת תמונה ל-[[יובל]]:

1. הוא מריץ `Glob: yuval/reference/*`.
2. אם יש קבצים — קורא 2-3 ומחלץ סגנון (פלטה, קומפוזיציה, סגנון אמנותי, mood).
3. מנסח prompt משולב של הבקשה + הסגנון.
4. קורא ל-[[gpt-image-gen (סקיל)]] ושומר ל-`yuval/outputs/`.
5. כותב sibling `.txt` עם ה-prompt.
6. מדווח ל-[[ראובן]] מה נוצר ובאיזה references השתמש.

## למי היא משויכת

- [[ראובן]] / המשתמש — מתחזק את `reference/`.
- [[יובל]] — כותב ל-`outputs/`, קורא מ-`reference/`.
